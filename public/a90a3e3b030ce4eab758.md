## はじめに

PCにつなぐたびにケーブルを挿したり外したり…そんな煩わしさから解放されたいなら、ScanSnapをNASに直接保存できるようにするのがおすすめです！
## 検証済み環境
* ScanSnap iX1600
* Ubuntu 22.04 + Samba 4.18
* クライアント: macOS
* ネットワーク: Wi-Fi

## 1. Samba共有フォルダの作成

まず、ScanSnap専用の共有ディレクトリを作ります。

```bash\sudo
sudo chmod 775 /opt/scansnap
```

次に、ユーザーを作成します。

```bash\sudo
useradd scansnap
passwd scansnap
smbpasswd -a scansnap
```

## 2. smb.confの設定

最低限以下の設定が必要です。`/etc/samba/smb.conf` に追記します。

```/etc/samba/smb.conf
[scansnap]
   path = /opt/scansnap
   browseable = yes
   guest ok = no
   valid users = scansnap
   read only = no
```

設定を反映します。

```bash
sudo systemctl restart smbd
```

## 3. ScanSnap側の設定

1. ScanSnap Home を起動します。
1. 新しいプロファイルを作成→ネットワークフォルダに保存を選択
3. 管理→保存先に以下を入力

```
smb://サーバーIP/scansnap
```

4. 認証情報に先ほど作った ユーザーとパスワードを入力
5. 接続

成功すれば保存先として登録できます。
## 4.完成！
お疲れまでした！
## まとめ

ScanSnapをNASに直接保存できるようにすると、パソコンが必要にならないので楽になります。家族や複数デバイスでデータをすぐ共有でき、扱いやすくなります。
